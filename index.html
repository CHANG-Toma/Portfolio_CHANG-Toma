Veille technologique

La sécurité des données Web pour un développeur

La veille technologique, est avant tout un état d'esprit dont il faut s'adopter; cela doit presque devenir une seconde nature. Il convient également d'être très au fait des canaux possibles d'information afin de les surveiller ou de savoir les utiliser les uns par rapport aux autres Pour la deuxième année de mon BTS, j'ai eu l'opportunité de sélectionner une technologie en veille sur le sujet des "technologies émergentes". En ce qui me concerne, j'ai orienté ma veille technologique vers l'importance de la sécurité des données Web pour un développeur, car cette thématique se marie de manière optimale avec ma spécialité de formation.

Introduction

Un site Web est devenu très accessible, depuis quelques années. Un domaine qui se développe très vite, mais très peu sont informés sur les vulnérabilités présentes dans leurs réalisations (Ou aussi appelées « failles »), un terme qui peut s’avérer très dangereux.

Typologie des applications Web

Une application Web est une extension dynamique d’un serveur Web. Une application web est formée d’un ensemble de composants web, de ressources statiques (sons, images, vidéos, …), de classes utilitaires et de bibliothèques.

Architecture

L’architecture est divisée en 3 grandes parties :
- Couche de présentation
- Couche métier
- Couche d’accès aux données

OWASP

L’OWASP, Open Web Application Security Project est un guide de sécurisation de 2001 des applications Web, c’est un « ouvrage » de référence des bonnes et mauvaises pratiques de développement, d’une base sérieuse en termes de statistique et en termes de ressource amenant à une base de réflexion sur la sécurité.

Risques des failles

L’injection SQL, est un type d’exploitation d’une faille de sécurité d’une application agissant avec une base donnée. L’attaquant détourne les requêtes en y injectant une chaîne non prévue par le développeur.
